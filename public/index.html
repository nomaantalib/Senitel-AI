Core Guardian Features

Preventing outages by proactively validating code and infrastructure before deployment.

Risk Analyzer

Scans Git branches, commits, active telemetry loads, and open Jira bugs to compute a predictive failure score before code is pushed.

Incident Time Machine

Reconstructs step-by-step chronology logs of previous production outages to audit cascading failure triggers and avoid recurrence.

Canary Release Advisor

Generates optimal traffic splits (10% -> 25% -> 50% -> 100%) and safety verification checks tailored to the active service health.

RCA Diagnostics

Correlates infrastructure anomalies in Prometheus logs with open code bugs using RAG search over incident runbooks.

How It Works

Behind the scenes of Sentinel's predictive decision-making engine.

01

Context Fetching

Sentinel queries GitHub (recent commit diffs), Jira (active blocker tickets), and Prometheus (CPU/Memory metrics).

02

RAG Runbook Search

The incident details are cross-referenced with your postmortem database (ChromaDB) to fetch similar failure templates.

03

Gemini Guardrail Synthesis

The context payload is processed by the Gemini model queue, auto-rotating to prevent free-tier limits and fallback key blockers.
